## thread_manager.gd `as` 运算符优先级 Bug 修复
- Bug: `"%.1f" % t.get(...) as float` 被解析为 `("str" % val) as float`（% 优先级高于 as）
- 修复: 第 113/114/248 行加括号 → `% (t.get(...) as float)` / `(as float) * 100`
- 3 行改动，纯语法修复

## manana_schema.gd required_canon 类型定义修复
- Bug: Schema 定义 `"required_canon": "string"`，但 Prompt 指导 LLM 输出 `["char_id"]` (数组)
- 结果: LLM 输出数组 → 验证器检测到 TYPE_ARRAY(28) → 报 "expected String, got 28"
- 修复: `manana_schema.gd:169` 改为 `"required_canon": "array"`

## interaction_pair.gd null char_ids 防御 + F8/空回车优化
- interaction_pair.gd:22: LLM 输出 char_ids 含 null → from_dict() 改为逐元素过滤 (只保留非空 String)
