## "400 input length too long" 根因诊断

- **不是 prompt 太大**：qwen3.5:9b 原生 256K 上下文，实际 prompt 仅 ~600-1000 字
- **真正原因：qwen3.5:9b 默认开启思考模式 (thinking/reasoning)**
  - `max_tokens=200` 时：思考吃掉全部 200 token，content 为空，finish_reason="length"
  - `max_tokens=1024` 时：思考~600 + 回复~400，正常输出
  - 关闭思考 (`reasoning_effort="none"`)：26 token 秒回

### 修复
- `max_tokens`: 200 → 2048（容纳思考+叙事输出）
- 增强错误日志：非200响应打印完整body，请求体也记录
- `_on_chat_completed` 新增 reasoning 非空但 content 为空的专项诊断提示
- `_build_characters_context()` 添加 2000 字安全截断
- 叙事历史从 5 条降至 3 条防止 prompt 膨胀

## Godot MCP Server 构建

- 为 Rain 项目构建了完整的 Godot MCP 应用，让 AI Agent 可以控制 Godot 4.6.3 引擎
- 架构: AI Agent ←→ stdio ←→ MCP Server (TypeScript) ←→ WebSocket ←→ Godot Editor Plugin
- 双通道设计: 文件系统通道 (始终可用) + WebSocket 通道 (需要编辑器运行)

### Godot Editor Plugin (`addons/godot_mcp/`)
- plugin.cfg: 插件元数据
- plugin.gd: EditorPlugin 入口，在编辑器启动时创建 MCPBridge 节点
- mcp_bridge.gd: WebSocket 服务器 (port 4099)，JSON-RPC 协议，16 个 API 方法
  - 场景/节点: get_scene_nodes, get_node_properties, set_node_property, create_node, delete_node, save_scene
  - 运行时: run_game, stop_game, is_game_running, call_autoload, get_autoload_list
  - 文件: read_project_file, write_project_file, list_project_files
  - 其他: get_editor_state, get_script_errors, execute_gdscript

### TypeScript MCP Server (`mcp-server/`)
- 28 个 MCP 工具，分 5 大类:
  - project_* (5): 项目文件读写、搜索
  - scene_* (7): 场景节点 CRUD
  - script_* (4): GDScript 读写、验证、列表
  - runtime_* (6): 游戏运行控制、autoload 调用
  - canon_* (10): Rain 专属 canon.json 管理
- GodotClient: WebSocket 客户端，请求超时 15s，连接超时 5s，自动 fallback 到 file-only 模式
- 编译通过 (TypeScript 5.9.3 + Node 22.22.2)，MCP tools/list 测试通过

## Godot MCP Addon 兼容性修复 (Godot 4.6.3)

- **问题**: 编辑器报 Parse Error，2 处 GDScript 语法错误
- **根因 1 (`mcp_bridge.gd:16`)**: `WebSocketServer` 类在 Godot 4 中已被移除
  - 修复: 改用 `TCPServer` + `WebSocketPeer` 手动构建 WebSocket 服务器
  - `_process`: `_server.poll()` + `get_connection_status()` → `_tcp_server.is_connection_available()`
  - `_accept_client`: `_server.take_peer()` → `_tcp_server.take_connection()` + `WebSocketPeer.accept_stream()`
- **根因 2 (`plugin.gd:13`)**: Godot 4.4+ 移除了 `GDScript.new()` 方法
  - 修复: `load("...").new()` → `Node.new()` + `set_script(bridge_script)`
- 两文件均已修复，无遗留 `WebSocketServer` 或 GDScript `.new()` 引用

## Godot MCP Addon 类型绕过修复

- **问题**: 运行时仍然报错 `plugin.gd:17 - Invalid assignment of property 'editor_interface'` 
- **根因**: `var _bridge: Node` → `_bridge.editor_interface = ...`，编译器从静态类型 `Node` 看不到动态脚本 `mcp_bridge.gd` 里的属性
- **修复**: 改用 `_bridge.set("editor_interface", get_editor_interface())` 绕过静态类型检查
- 结论: Godot 4.4+ 中 `Node.new() + set_script()` 后，动态属性访问需要用 `set()` / `get()` 方法
